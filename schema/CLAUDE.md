# Schema マイグレーションガイド

## ファイル命名規則

- 形式: `XXX_description.sql`（例: `007_add_calendar_datetime_fields.sql`）
- XXX: 3桁の連番（001から開始、既存の最大番号 + 1）
- description: スネークケースで機能を表現

## マイグレーションファイルの書き方

- 既存のマイグレーションファイルは変更せず、必ず新しい連番ファイルを追加する（適用済みDBとの整合が壊れるため）
- 冪等に書く（`IF NOT EXISTS` を使用）
- 既存データの移行が必要な場合は同一ファイル内でUPDATE文も記述
- コメントで変更内容を説明

## 追随作業と本番適用

スキーマ変更の追随作業（shared型・リポジトリ・テスト・`docs/er-diagram.md`）と、
本番DBへの適用手順（ユーザーの明示的な指示があるときのみ・バックアップ必須）は **db-migration スキル** が唯一の正。
このディレクトリでSQLを書く前に、必ず同スキルのチェックリストを確認すること。
