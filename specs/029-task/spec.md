**Created**: 2025-11-18
**Status**: Draft

## Clarifications

### Session 2025-11-18

- Q: コピー成功時の視覚的フィードバックの具体的な実装方法はどれが望ましいですか？ → A: ボタンアイコンが一時的に変化（例：クリップボード→チェックマーク、2秒間表示後に戻る）
- Q: クリップボードAPIが利用できない環境でのフォールバック動作は具体的にどうすべきですか？ → A: 何もしない（ボタンは表示されるが、クリックしても無反応）、ただしconsole.logにはエラーを出力
- Q: コピーボタンの配置位置はどこが最適ですか？ → A: 本文エリアの右上隅（コンテンツ表示エリア内）
- Q: 「すべてコピー」ボタンの配置位置はどこが最適ですか？ → A: ページヘッダーエリア（タイトル・ステータス・ブックマークボタンの横）
- Q: コピー成功時のアイコン変化（チェックマーク表示）の表示時間は何秒が最適ですか？ → A: 1秒間（素早く元に戻る）

## User Scenarios & Testing *(mandatory)*

### User Story 1 - Copy Task/Memo Body as Markdown (Priority: P1)

ユーザーは、タスクやメモの本文をMarkdown形式でコピーして、AIツールや他のアプリケーションに貼り付けたい。現在はEdit画面に入らないとMarkdown形式のテキストを取得できず、特にスマートフォンでの操作が不便である。

**Why this priority**: これは最も基本的で重要な機能です。ユーザーがMarkdown形式でコンテンツをコピーできることで、AIツールとの連携や他のドキュメントへの転記が簡単になります。

**Independent Test**: タスク詳細画面を開き、コピーボタンをクリックして、クリップボードにMarkdown形式のテキストがコピーされることを確認する。コピーしたテキストをテキストエディタに貼り付けて、元のMarkdown形式が保持されていることを検証できる。

**Acceptance Scenarios**:

1. **Given** タスク詳細画面を表示している, **When** 本文エリアのコピーボタンをクリックする, **Then** 本文のMarkdownテキストがクリップボードにコピーされる
2. **Given** メモ詳細画面を表示している, **When** 本文エリアのコピーボタンをクリックする, **Then** 本文のMarkdownテキストがクリップボードにコピーされる
3. **Given** コピーボタンをクリックした, **When** テキストエディタに貼り付ける, **Then** Markdown形式のテキスト（レンダリング前のrawテキスト）が貼り付けられる
4. **Given** 本文が空のタスク/メモを表示している, **When** コピーボタンをクリックする, **Then** 空文字列がクリップボードにコピーされる（エラーは発生しない）

---

### User Story 2 - Copy Individual Comment as Markdown (Priority: P2)

ユーザーは、タスクやメモに追加された個々のコメントをMarkdown形式でコピーしたい。複数のコメントがある場合、特定のコメントだけを選択してコピーできる必要がある。

**Why this priority**: コメントは追加情報や議論の履歴を含むため、それぞれを個別にコピーできることで、必要な情報だけを抽出して利用できます。P1の機能が動作すれば、同じ仕組みでコメントにも適用できます。

**Independent Test**: タスク詳細画面のコメントセクションで、各コメントに表示されたコピーボタンをクリックし、そのコメントのMarkdownテキストだけがクリップボードにコピーされることを確認する。

**Acceptance Scenarios**:

1. **Given** コメントが1つ以上あるタスク/メモを表示している, **When** 特定のコメントのコピーボタンをクリックする, **Then** そのコメントのMarkdownテキストだけがクリップボードにコピーされる
2. **Given** 複数のコメントがあるタスク/メモを表示している, **When** 異なるコメントのコピーボタンを順次クリックする, **Then** 各クリックで対応するコメントのMarkdownテキストがクリップボードにコピーされる
3. **Given** コメントのコピーボタンをクリックした, **When** テキストエディタに貼り付ける, **Then** Markdown形式のコメントテキストが貼り付けられる

---

### User Story 3 - Copy All Content as Markdown (Priority: P3)

ユーザーは、タスク/メモの本文とすべてのコメントを含む完全な内容を、一度の操作でMarkdown形式でコピーしたい。これにより、AIツールに全体のコンテキストを渡したり、完全なバックアップを取得したりできる。

**Why this priority**: 一括コピー機能は便利だが、P1とP2が実装されていれば手動で複数回コピーすることで同様の結果を得られるため、優先度は低い。ただし、ユーザー体験を向上させる付加価値がある。

**Independent Test**: タスク詳細画面で「すべてコピー」ボタンをクリックし、タイトル・本文・すべてのコメントを含む構造化されたMarkdownテキストがクリップボードにコピーされることを確認する。

**Acceptance Scenarios**:

1. **Given** 本文とコメントがあるタスク/メモを表示している, **When** 「すべてコピー」ボタンをクリックする, **Then** タイトル、本文、すべてのコメントが構造化されたMarkdown形式でクリップボードにコピーされる
2. **Given** コメントがないタスク/メモを表示している, **When** 「すべてコピー」ボタンをクリックする, **Then** タイトルと本文だけがMarkdown形式でクリップボードにコピーされる
3. **Given** 「すべてコピー」でコピーした内容を, **When** テキストエディタに貼り付ける, **Then** 以下の構造でMarkdownが貼り付けられる:
   ```
   # [タイトル]

   [本文]

   ## Comments

   ### Comment 1 ([作成日時])
   [コメント1の内容]

   ### Comment 2 ([作成日時])
   [コメント2の内容]
   ```

---

### Edge Cases

- コピーボタンをクリックしたがクリップボードAPIが利用できない環境の場合、どう対処するか？（フォールバック: テキストを選択状態にして手動コピーを促す、またはエラーメッセージを表示）
- 本文やコメントに特殊文字やエスケープが必要な文字が含まれている場合、正しくコピーされるか？（rawテキストをそのままコピーするため、問題なし）
- モバイルデバイス（iOS Safari、Android Chrome）でクリップボードAPIが正しく動作するか？
- コピーボタンを連続で高速クリックした場合、正しく動作するか？（デバウンス処理が必要か？）
- ネットワークエラーや遅延がある場合でも、ローカルに表示されているデータをコピーできるか？（できるべき - クライアント側のデータをコピー）

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: システムは、タスク/メモ詳細画面の本文エリアの右上隅（コンテンツ表示エリア内）にMarkdownコピーボタンを表示しなければならない
- **FR-002**: システムは、各コメントの右上隅（コンテンツ表示エリア内）にMarkdownコピーボタンを表示しなければならない
- **FR-003**: システムは、タスク/メモ詳細画面のページヘッダーエリア（タイトル・ステータス・ブックマークボタンの横）に「すべてコピー」ボタンを表示しなければならない
- **FR-004**: コピーボタンは、コードブロックのコピーボタンに一般的に使用されるアイコン（例：重なった四角形、クリップボードアイコン）を使用しなければならない（絵文字ではない）
- **FR-005**: 本文のコピーボタンをクリックすると、本文のMarkdown rawテキストがクリップボードにコピーされなければならない
- **FR-006**: コメントのコピーボタンをクリックすると、そのコメントのMarkdown rawテキストがクリップボードにコピーされなければならない
- **FR-007**: 「すべてコピー」ボタンをクリックすると、タイトル・本文・すべてのコメントが構造化されたMarkdown形式でクリップボードにコピーされなければならない
- **FR-008**: コピー操作が成功した場合、ボタンアイコンを一時的に変化させる視覚的なフィードバック（クリップボードアイコン→チェックマークアイコン、1秒間表示後に元に戻る）をユーザーに提供しなければならない
- **FR-009**: クリップボードAPIが利用できない環境では、コピーボタンは表示されるがクリックしても無反応とし、エラー情報をconsole.logに出力しなければならない
- **FR-010**: コピー機能は、デスクトップとモバイルブラウザの両方で動作しなければならない
- **FR-011**: コピーされるテキストは、レンダリング後のHTMLではなく、元のMarkdown形式のrawテキストでなければならない
- **FR-012**: 「すべてコピー」機能では、以下の構造でMarkdownを生成しなければならない:
  - タイトル（H1見出し）
  - 本文（元のMarkdown）
  - コメントセクション（H2見出し "Comments"）
  - 各コメント（H3見出し + 作成日時 + 本文）

### Key Entities

このフィーチャーは既存のデータモデル（Task、Memo、Comment）を利用し、新しいエンティティは追加しません。UIレイヤーでの機能追加のみです。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: ユーザーは、タスク/メモの本文を3クリック以内（詳細画面を開く→コピーボタンをクリック）でMarkdown形式でコピーできる
- **SC-002**: コピー操作の成功率が95%以上（主要ブラウザ: Chrome、Safari、Firefox、モバイルSafari、モバイルChromeで検証）
- **SC-003**: コピーボタンをクリックしてからクリップボードにテキストがコピーされるまでの時間が200ミリ秒以内
- **SC-004**: コピーされたMarkdownテキストを他のアプリケーション（テキストエディタ、AIツール）に貼り付けた際、元のMarkdown形式が100%保持されている
- **SC-005**: モバイルデバイスでのコピー操作の成功率が90%以上（iOS Safari、Android Chrome）
- **SC-006**: ユーザーがEdit画面に入らずに済むことで、Markdownコピーのための操作ステップが50%削減される（Edit画面を開く→テキストを選択→コピー の3ステップが、コピーボタンクリックの1ステップに）
