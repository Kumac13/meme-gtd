# Feature Specification: タスクをメモにdemote（コピー）する機能

**Feature Branch**: `001-demote`
**Created**: 2025-11-29
**Status**: Draft
**Input**: User description: "タスクをメモにdemote（コピー）する機能"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - タスク完了時にドキュメントとしてメモにコピー (Priority: P1)

調査や検討プロセスを含むタスクを完了した際、その内容をドキュメントとして残すためにメモとしてコピーを作成する。タスク自体はDoneにマークしつつ、調査結果や考察をメモとして保存できる。

**Why this priority**: タスクの成果物をドキュメント化するという本機能の核心的なユースケース。これがないと機能の存在意義がない。

**Independent Test**: CLIまたはWeb UIからタスクを選択し、demoteを実行して新規メモが作成されることを確認できる。

**Acceptance Scenarios**:

1. **Given** 調査内容を含むタスクが存在する, **When** demoteを実行する, **Then** タスクのタイトル・本文・コメントが1つのメモとして作成される
2. **Given** タスクにラベルとプロジェクトが設定されている, **When** demoteを実行する, **Then** 新規メモに同じラベルとプロジェクトが継承される
3. **Given** demoteが完了した, **When** 元タスクを確認する, **Then** 元タスクは変更されていない（削除もステータス変更もされない）

---

### User Story 2 - demote前に内容を編集 (Priority: P2)

demote実行時にエディタが開き、メモの内容を確認・編集してから保存できる。不要な部分を削除したり、追加のコメントを書き加えたりできる。

**Why this priority**: 自動生成された内容をそのまま使うだけでなく、ユーザーが内容を調整できることで実用性が高まる。

**Independent Test**: demoteコマンド実行時にエディタが開き、編集後の内容でメモが作成されることを確認できる。

**Acceptance Scenarios**:

1. **Given** タスクを選択してdemoteを実行する, **When** エディタが開く, **Then** タイトル・本文・コメントが自動的に組み立てられた状態で表示される
2. **Given** エディタで内容を編集する, **When** 保存して終了する, **Then** 編集後の内容でメモが作成される
3. **Given** エディタで内容を確認する, **When** 保存せずに終了する, **Then** メモは作成されない

---

### User Story 3 - 元タスクへのリンクで関連を追跡 (Priority: P2)

作成されたメモから元タスクへのリンクが自動的に作成され、どのタスクから派生したメモかを後から追跡できる。

**Why this priority**: ドキュメントとタスクの関連性を維持することで、後から文脈を辿れる。

**Independent Test**: demoteで作成されたメモのリンク情報を確認し、元タスクへの参照が存在することを確認できる。

**Acceptance Scenarios**:

1. **Given** demoteでメモを作成する, **When** メモのリンク情報を確認する, **Then** 元タスクへの`derived_from`リンクが存在する
2. **Given** 元タスクへのリンクがある, **When** リンクを辿る, **Then** 元タスクの詳細を確認できる

---

### Edge Cases

- タスクにタイトルのみで本文がない場合はどうなるか？ → タイトルのみでメモを作成する
- タスクにコメントがない場合はどうなるか？ → コメントセクションを省略する
- 既に削除されたタスクをdemoteしようとした場合はどうなるか？ → エラーを返す
- demote実行中にエディタがクラッシュした場合はどうなるか？ → メモは作成されない

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: システムはタスクのタイトル・本文・コメントを1つのメモ本文に組み立てる機能を提供しなければならない
- **FR-002**: システムはdemote時に元タスクを変更してはならない（削除・ステータス変更禁止）
- **FR-003**: システムは新規メモから元タスクへの`derived_from`リンクを自動作成しなければならない
- **FR-004**: システムはタスクのラベルを新規メモに継承しなければならない
- **FR-005**: システムはタスクのプロジェクトを新規メモに継承しなければならない
- **FR-006**: CLIはdemote実行前にエディタを開いて内容編集を可能にしなければならない
- **FR-007**: CLIは`--no-editor`オプションで編集をスキップできなければならない
- **FR-008**: APIは`POST /api/tasks/:id/demote`エンドポイントを提供しなければならない
- **FR-009**: Web UIはタスク詳細画面から「メモにコピー」アクションを実行できなければならない

### Key Entities

- **Task**: demoteの対象。タイトル、本文、コメント、ラベル、プロジェクトを持つ
- **Memo**: demoteの結果として作成される。タイトルフィールドは持たず、本文のみ
- **Link**: MemoからTaskへの`derived_from`関連を表す
- **Label**: タスクからメモに継承される分類タグ
- **Project**: タスクからメモに継承されるプロジェクト所属

### メモ本文の構成

demote時に以下の形式でメモ本文が組み立てられる：

```
# {task.title}

{task.body_md}

---
## コメント

### {comment.created_at}
{comment.body_md}

### {comment.created_at}
{comment.body_md}
```

- タイトルは見出し1（`#`）として本文の先頭に追加
- 本文はタイトルの後に追加
- コメントは時系列順で本文の後に追加
- コメントがない場合は「コメント」セクション自体を省略

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: ユーザーはタスクからメモへのdemoteを3クリック以内（またはCLIで1コマンド）で完了できる
- **SC-002**: demoteで作成されたメモは元タスクの全ての内容（タイトル・本文・コメント）を含む
- **SC-003**: demoteで作成されたメモは元タスクのラベルとプロジェクトを100%継承する
- **SC-004**: demote後も元タスクは完全に保持され、いかなる変更も加えられない
- **SC-005**: ユーザーは作成されたメモから元タスクへ1クリックで遷移できる

## Assumptions

- エディタはシステムのデフォルトエディタ（`$EDITOR`環境変数）を使用する
- コメントの日時フォーマットは既存のシステム設定に従う
- demoteはタスクの現在の状態（Done含む）に関係なく実行可能
