# Data Model: Fuzzy Search Extension

**Feature**: 025-a - Fuzzy Search for Tasks and Memos
**Date**: 2025-11-04

## Overview

This feature **does not introduce new database entities**. It extends existing query interfaces to support free-text search using the existing FTS5 infrastructure.

---

## Existing Database Schema (No Changes)

### Core Table: `issues`

The existing `issues` table stores both tasks and memos:

```sql
CREATE TABLE issues (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  type TEXT NOT NULL CHECK (type IN ('task', 'memo')),
  title TEXT,
  body_md TEXT NOT NULL,
  status TEXT CHECK (status IN ('open', 'next', 'waiting', 'scheduled', 'done', 'canceled')),
  scheduled_on TEXT,
  meta TEXT DEFAULT '{}',
  created_at TEXT NOT NULL,
  updated_at TEXT NOT NULL,
  is_bookmarked INTEGER DEFAULT 0 NOT NULL,
  is_deleted INTEGER DEFAULT 0 NOT NULL
);
```

### FTS5 Virtual Table: `issues_fts`

Already exists (schema/001_init.sql:97-103):

```sql
CREATE VIRTUAL TABLE issues_fts USING fts5(
  title, body_md,
  content=issues,
  content_rowid=id,
  tokenize='unicode61'
);
```

**No schema migration required** - FTS5 infrastructure is already in place.

## Frontend Query Model

### ParsedSearchQuery (packages/web/src/utils/queryParser.ts)

```typescript
export interface ParsedSearchQuery {
  labels?: string[];
  status?: string;
  freeText?: string;  // 🆕 NEW: Extracted free-text search terms
  rawQuery?: string;
}
```

**New field**:
- `freeText?: string`: Text portion of query after removing all `key:value` patterns
  - Example: `"label:bug status:open login screen"` → `freeText: "login screen"`
  - Passed to API as `search` parameter
  - Not parsed in link search context (clarification Q5)

**Parsing logic**:

```typescript
export function parseSearchQuery(query: string): ParsedSearchQuery {
  if (!query || query.trim() === '') {
    return {};
  }

  const result: ParsedSearchQuery = {
    rawQuery: query,
  };

  // Extract label: filters (existing logic)
  const labelMatches = query.match(/label:([^\s]+)/g);
  if (labelMatches) {
    // ... existing code ...
  }

  // Extract status: filter (existing logic)
  const statusMatch = query.match(/status:([^\s,]+)/);
  if (statusMatch) {
    result.status = statusMatch[1];
  }

  // 🆕 NEW: Extract free-text (remove all key:value patterns)
  const freeText = query
    .replace(/\w+:[^\s]+/g, '')  // Remove structured filters
    .trim();

  if (freeText) {
    result.freeText = freeText;
  }

  return result;
}
```

## Validation Rules

### Search Query Validation

**Minimum length** (link search only):
- Rule: At least 3 characters required for link search
- Rationale: Prevent overly broad searches, reduce server load
- Applied in: AddLinkInline.tsx (frontend validation)

**Special characters**:
- Rule: Special characters treated as literal (no escaping needed)
- Supported: `@`, `.`, `+`, `#`, etc.
- Rationale: FTS5 tokenizer handles special chars correctly

**Case sensitivity**:
- Rule: Case-insensitive matching
- Implementation: FTS5 `unicode61` tokenizer provides automatic case folding
- Applies to: All Unicode characters (Latin, Japanese, emoji)

### Result Limits

**Link search results**:
- Maximum: 20 results displayed
- Overflow: Show "More results available - refine your search" message
- Implementation: `SearchResults.tsx` slices array to `maxResults`

**Main list search results**:
- No specific limit (returns all matches)
- Existing pagination/limit parameters still apply
- Performance: FTS5 handles 10K rows in <50ms

## Security Considerations

### XSS Risk (Preview Field)

**Risk**: `preview` field contains HTML markers (`<mark>...</mark>`)

**Mitigation**:
1. Content is markdown-only (no user-provided HTML)
2. Frontend uses `dangerouslySetInnerHTML` (controlled risk)
3. Consider sanitization library (e.g., DOMPurify) if HTML input ever allowed

**Current assessment**: Low risk (markdown content only).

### SQL Injection

**Risk**: FTS5 MATCH queries with user input

**Mitigation**:
1. ✅ All queries use parameterized statements (`@search` placeholder)
2. ✅ FTS5 tokenizer escapes special characters automatically
3. ✅ No dynamic SQL string concatenation

**Current assessment**: Protected (parameterized queries).
