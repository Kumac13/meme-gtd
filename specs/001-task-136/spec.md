# Feature Specification: 画像添付機能

**Feature Branch**: `001-task-136`
**Created**: 2025-12-05
**Status**: Draft
**Input**: Task #136: 画像添付機能 - スクリーンショットをメモ/タスクに貼り付け、Claude Codeで認識可能な形式で出力

## User Scenarios & Testing *(mandatory)*

### User Story 1 - Web UIから画像をアップロード (Priority: P1)

ユーザーがWeb UI上でメモやタスクを編集中に、スクリーンショットや画像ファイルをドラッグ＆ドロップまたはファイル選択でアップロードし、本文に画像参照を挿入する。

**Why this priority**: 画像添付の最も一般的なユースケース。Web UIでの編集中に視覚的な情報を追加できることが、この機能の核心的価値。

**Independent Test**: Web UIでメモを開き、画像をドラッグ＆ドロップして本文に挿入。保存後、画像が表示されることを確認。

**Acceptance Scenarios**:

1. **Given** ユーザーがWeb UIでメモ編集画面を開いている, **When** PNG画像をエディタにドラッグ＆ドロップする, **Then** 画像がサーバーに保存され、Markdown形式の画像参照 `![alt](絶対パス)` がエディタに挿入される
2. **Given** ユーザーがWeb UIでタスク編集画面を開いている, **When** ファイル選択ダイアログから画像を選択する, **Then** 画像がサーバーに保存され、本文に画像参照が挿入される
3. **Given** ユーザーがWeb UIでメモを閲覧している, **When** 本文にMarkdown画像参照が含まれている, **Then** 画像がインライン表示される

---

### User Story 2 - CLI出力をClaude Codeにコピペ (Priority: P2)

ユーザーがCLIでメモやタスクを表示し、その出力をそのままClaude Codeに貼り付けて相談する。画像パスはClaude Codeが認識できる絶対パス形式で出力される。

**Why this priority**: mgtdの出力をClaude Codeに渡して相談するという主要ユースケースを支える。画像アップロード機能がなくても、既存の画像参照を正しく出力できれば価値がある。

**Independent Test**: 画像参照を含むメモをCLIで表示し、出力をClaude Codeにコピペして画像パスが認識されることを確認。

**Acceptance Scenarios**:

1. **Given** 画像参照 `![スクショ](/Users/user/.mgtd/attachments/abc123.png)` を含むメモが存在する, **When** `mgtd memo show 42` を実行する, **Then** Markdown本文がそのまま出力され、画像パスは絶対パスで表示される
2. **Given** 画像参照を含むタスクが存在する, **When** `mgtd task show 10` を実行する, **Then** 画像パスを含むMarkdown本文がそのまま出力される

---

### Edge Cases

- ユーザーが対応していないファイル形式（例: .bmp, .tiff）をアップロードしようとした場合は、エラーメッセージを表示し受け付けない
- アップロードファイルサイズが上限（10MB）を超えた場合は、エラーメッセージを表示する
- 保存先ディレクトリが存在しない場合は、自動的に作成する
- 画像ファイルが削除・移動されて存在しない場合、Web UIでは代替テキスト（alt）とエラー表示、CLIでは絶対パスをそのまま出力
- 同じissueに複数の画像を添付した場合、それぞれユニークなファイル名（UUID）で保存される

## Requirements *(mandatory)*

### Functional Requirements

#### ストレージ

- **FR-001**: システムは画像ファイルを `~/.mgtd/attachments/{uuid}.{ext}` の形式で保存しなければならない（フラット構造、issueに紐付けない）
- **FR-002**: 画像パスは常に絶対パス形式で保存・参照されなければならない

#### 対応フォーマット

- **FR-003**: システムは以下の画像形式をサポートしなければならない: PNG, JPEG, GIF, WebP
- **FR-004**: システムはアップロード時にファイルサイズを検証し、10MBを超えるファイルは拒否しなければならない

#### Web UI

- **FR-005**: ユーザーはWeb UIでドラッグ＆ドロップまたはファイル選択により画像をアップロードできなければならない
- **FR-006**: システムは画像アップロード後、Markdown形式の画像参照 `![filename](絶対パス)` をエディタに挿入しなければならない
- **FR-007**: システムは画像配信用のエンドポイント `GET /api/attachments/:filename` を提供しなければならない
- **FR-008**: Markdownレンダリング時、絶対パス形式の画像参照をAPIエンドポイントURLに変換して表示しなければならない

#### CLI

- **FR-009**: CLIの `memo show` および `task show` コマンドは、本文内の画像参照を絶対パス形式のまま出力しなければならない

### Key Entities

- **Attachment**: 添付された画像ファイルを表す。ファイル名（UUID形式）、拡張子、保存先絶対パスを持つ。特定のissueには紐付かず、フラットなディレクトリ構造で管理される
- **Issue (既存)**: メモまたはタスク。本文（body）にMarkdown形式の画像参照を含むことができる

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: ユーザーはWeb UIから10秒以内に画像をアップロードし、エディタに画像参照を挿入できる
- **SC-002**: アップロードされた画像はWeb UI上で即座に表示される
- **SC-003**: CLI出力をコピペした際、Claude Codeが画像パスを認識できる絶対パス形式で出力される
- **SC-004**: 対応形式（PNG, JPEG, GIF, WebP）の画像を100%正常に保存・表示できる

## Assumptions

- 画像のメタデータ（サイズ、作成日時等）はデータベースで管理せず、ファイルシステム上のファイルのみで管理する
- 画像の圧縮・リサイズは行わず、オリジナルファイルをそのまま保存する
- サムネイル生成は行わない
- issue削除時の画像ファイル連動削除は将来の課題とし、このスコープでは対応しない
- CLIからの画像添付機能はスコープ外とする（Web UIのみ）

## Out of Scope

- CLIからの画像添付コマンド
- 画像の圧縮・リサイズ機能
- サムネイル自動生成
- 画像メタデータのDB管理
- issue削除時の画像ファイル連動削除
- 画像のコピー（クリップボードへの画像データコピー）
- 保存先のカスタマイズ（デフォルトパス `~/.mgtd/attachments/` のみ使用）
