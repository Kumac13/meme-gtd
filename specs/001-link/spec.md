# Feature Specification: link設定時の検索機能

**Feature Branch**: `001-link`
**Created**: 2025-11-30
**Status**: Draft
**Input**: User description: "link設定時に検索ができるようにする"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - タイトル検索によるリンク先選択 (Priority: P1)

ユーザーがタスクまたはメモにリンクを追加する際、リンク先のタイトルをテキスト検索して候補リストから選択できる。現状はIDを手動入力する必要があり、別タブでIDを確認する手間が発生している。

**Why this priority**: リンク設定の根本的な課題（ID確認の手間）を解決する中核機能。これがなければ機能改善の意味がない。

**Independent Test**: 検索テキストを入力し、マッチするタスク/メモが候補リストに表示されること。候補をクリックしてリンクが作成されることで検証可能。

**Acceptance Scenarios**:

1. **Given** リンク追加フォームが表示されている状態、**When** 検索フィールドにタイトルの一部を入力する、**Then** 入力文字列にマッチするタスクとメモが候補リストに表示される
2. **Given** 検索結果が表示されている状態、**When** 候補リストからアイテムをクリックする、**Then** そのアイテムがリンク先として選択される
3. **Given** 検索結果が表示されている状態、**When** 検索文字列を変更する、**Then** 候補リストがリアルタイムで更新される

---

### User Story 2 - 初期表示での最近のアイテム表示 (Priority: P2)

検索入力前に、最近のタスクとメモが候補として初期表示される。これにより、検索せずに直接選択することも可能。

**Why this priority**: 検索の手間を省略し、よく使うアイテムへの素早いアクセスを提供。P1の補完機能。

**Independent Test**: リンク追加フォームを開いた時点で、最近のタスク/メモが表示されていることを確認。

**Acceptance Scenarios**:

1. **Given** リンクタイプを選択した状態、**When** 検索/選択UIが表示される、**Then** 最近のタスクとメモが10件まで初期表示される
2. **Given** 初期表示状態、**When** 候補をクリックする、**Then** 検索せずに直接選択できる

---

### User Story 3 - キーボードナビゲーション (Priority: P3)

矢印キーで候補リストを移動し、Enterキーで選択を確定できる。マウスを使わずに素早く操作可能。

**Why this priority**: 操作効率を向上させるが、マウス操作でも目的は達成できるため優先度を下げる。

**Independent Test**: キーボードのみで候補選択が完了できることを確認。

**Acceptance Scenarios**:

1. **Given** 候補リストが表示されている状態、**When** ↓キーを押す、**Then** フォーカスが次の候補に移動する
2. **Given** 候補にフォーカスがある状態、**When** ↑キーを押す、**Then** フォーカスが前の候補に移動する
3. **Given** 候補にフォーカスがある状態、**When** Enterキーを押す、**Then** その候補がリンク先として選択される
4. **Given** 候補リストが表示されている状態、**When** Escキーを押す、**Then** 選択がキャンセルされUIが閉じる

---

### Edge Cases

- 検索結果が0件の場合、「該当するアイテムがありません」と表示する
- 自分自身（編集中のタスク/メモ）は候補から除外される
- 既にリンク済みのアイテムは候補に表示されるが、重複追加時はエラーを表示する
- 入力中に300ms以上入力がない場合に検索を実行する（デバウンス）
- 検索中はローディング表示を行う

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: システムは検索フィールドへの入力に基づき、タスクとメモをタイトルで検索できなければならない
- **FR-002**: システムは検索結果を候補リストとして最大10件まで表示しなければならない
- **FR-003**: ユーザーは候補リストからクリックまたはキーボード操作でアイテムを選択できなければならない
- **FR-004**: システムは検索入力前に最近のタスク/メモを10件まで初期表示しなければならない
- **FR-005**: システムは現在編集中のアイテム自身を候補から除外しなければならない
- **FR-006**: システムはキーボードナビゲーション（↑↓キーで移動、Enterで選択、Escでキャンセル）をサポートしなければならない
- **FR-007**: システムは検索入力に対して300msのデバウンスを適用し、過剰なリクエストを防止しなければならない
- **FR-008**: 候補リストには各アイテムのID、タイプ（Task/Memo）、タイトル、ステータスを表示しなければならない

### Key Entities

- **Issue（タスク/メモ）**: 検索対象となるエンティティ。ID、タイプ、タイトル、ステータスを持つ
- **Link**: タスク/メモ間の関連付け。ソースID、ターゲットID、リンクタイプ（parent/child/relates/derived_from）を持つ

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: ユーザーはリンク先を5秒以内に検索・選択できる（従来のID確認→入力の手間を削減）
- **SC-002**: 検索結果は入力完了から1秒以内に表示される
- **SC-003**: キーボードのみでリンク設定操作を完了できる
- **SC-004**: 既存のリンク追加フローを維持しつつ、ID直接入力の代わりに検索選択UIを提供する

## Assumptions

- 既存の検索API（`/api/tasks?search=`, `/api/memos?search=`）をそのまま活用する
- 検索はタイトルに対する部分一致で行う（既存APIの仕様）
- 「最近のアイテム」は更新日時の降順で取得する
- タスク作成フォーム（TaskFormLinks）と既存アイテム編集（AddLinkInline）の両方で同じ検索UIを使用する

## Scope

### In Scope

- リンク追加時の検索/選択UI（IssuePickerコンポーネント）
- タスク作成フォームへの適用
- 既存アイテム詳細ページへの適用
- キーボードナビゲーション

### Out of Scope

- 検索APIの変更
- リンクタイプの追加・変更
- バックエンド側の変更
