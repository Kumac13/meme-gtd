# ドキュメントガイド

## ドキュメント一覧

### `architecture.md` - アーキテクチャガイド

リポジトリ全体の構造と変更の波及範囲を示す「地図」。

**書くべき内容**:
- パッケージ依存関係と各パッケージの責務
- API契約の同期チェーン（Zod → openapi.yaml → Webクライアント → iOS Swiftモデル）
- 変更種別ごとの影響マップ
- 設計意図（なぜそうなっているか）

**参照タイミング**: 実装開始前（必須）、変更の影響範囲を判断するとき
**更新タイミング**: パッケージ追加・アーキテクチャ変更時

### `requirement.md` - 要件定義

プロダクト全体の要件を定義する最重要ドキュメント。

**書くべき内容**:
- プロダクト概要・目的・対象ユーザー
- スコープと前提条件
- データモデル（エンティティ、フィールド定義）
- 機能要件

**参照タイミング**: 実装開始前（必須）
**更新タイミング**: 機能追加・変更時

### `cli-commands.md` - CLIコマンドリファレンス

CLIコマンドの使い方を記載するリファレンス。

**書くべき内容**:
- コマンド構文とオプション
- 使用例（省略せず全体を記載）
- フィルタリング・検索の使い方

**参照タイミング**: CLIコマンド追加時
**更新タイミング**: CLIコマンド追加・変更時

### `api-filtering.md` - APIフィルタリング仕様

REST APIのクエリパラメータ仕様を記載。

**書くべき内容**:
- エンドポイント一覧
- クエリパラメータの定義
- リクエスト・レスポンス例

**参照タイミング**: API変更時
**更新タイミング**: API変更時

### `versioning.md` - バージョン管理戦略

SemVerルールとリリースプロセスを定義。

**書くべき内容**:
- バージョニング方針（Fixed Versioning）
- MAJOR/MINOR/PATCHの判断基準
- リリース手順

**参照タイミング**: リリース前
**更新タイミング**: ルール変更時のみ

### `operations.md` - 運用管理ガイド

運用課題の整理・改善ロードマップ・運用手順（Runbook）。

**書くべき内容**:
- 本番/テスト環境の定義と環境変数一覧
- バックアップ・リストア・ヘルスチェック・systemd 常駐化の手順
- 運用改善のフェーズ別ロードマップ

**参照タイミング**: サーバー運用・障害対応時、運用系機能の実装前
**更新タイミング**: 運用系機能の追加・変更時

## ルートドキュメントの更新

| ドキュメント | 更新タイミング |
|-------------|----------------|
| `README.md` | 機能追加・変更時 |
| `README.ai.md` | スキーマ変更・新機能追加時（他AI向け） |
| `CHANGELOG.md` | リリース前 |

## 書き方ルール

- 言語: 日本語（開発者向けドキュメント）
- フォーマット: Markdown
- コード例は省略せず全体を記載
