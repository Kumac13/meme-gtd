# Web記事保存用 ブラウザ拡張機能 セットアップガイド

> 目的: Chrome拡張機能のビルド・インストール・使用手順（人間向け）
> 読むタイミング: ブラウザ拡張の環境構築・トラブルシューティング時
> 更新タイミング: 拡張のビルド方法・設定方法の変更時

meme-gtdには、閲覧中のWebページをワンクリックでローカルデータベースに保存するためのブラウザ拡張機能（Chrome/Chromium用）が付属しています。

## 前提条件

- Node.js 22 以上（`.tool-versions` 参照）
- pnpm 10 系（`package.json` の `packageManager` に固定）
- Google Chrome または Chromiumベースのブラウザ（Edge, Brave等）

## ビルド手順

拡張機能を使用するには、まずソースコードからビルドする必要があります。

1. **プロジェクトルートで依存関係をインストール**
   ```bash
   pnpm install
   ```

2. **拡張機能のビルド**
   プロジェクトルートで以下のコマンドを実行します：
   ```bash
   pnpm --filter @meme-gtd/extension build
   ```
   または、`packages/extension` ディレクトリに移動して実行します：
   ```bash
   cd packages/extension
   pnpm build
   ```

   成功すると、`packages/extension/dist` ディレクトリにビルド成果物が生成されます。

## ブラウザへのインストール手順

1. **拡張機能管理ページを開く**
   ChromeのURLバーに `chrome://extensions` と入力してエンターキーを押します。

2. **デベロッパーモードを有効化**
   画面右上の「デベロッパーモード」トグルスイッチをONにします。

3. **パッケージ化されていない拡張機能を読み込む**
   画面左上に現れる「パッケージ化されていない拡張機能を読み込む」ボタンをクリックします。

4. **ディレクトリを選択**
   ファイル選択ダイアログで、先ほどビルドした `packages/extension/dist` ディレクトリを選択します。
   ※ `packages/extension` 自体ではなく、その中の `dist` ディレクトリを選択してください。

5. **インストールの確認**
   拡張機能リストに「Meme GTD Saver」が表示されていればインストール成功です。

## 使用方法

### 1. API接続先の設定（初回のみ・または環境変更時）

拡張機能はローカルのAPIサーバーと通信します。デフォルトでは `http://localhost:3000` に接続しますが、開発環境やテスト環境（ポート3001など）を使用する場合は設定変更が必要です。

1.  ブラウザのツールバーで「Meme GTD Saver」アイコンを右クリックし、「オプション」を選択します。
2.  「API URL」フィールドに、接続したいAPIサーバーのURLを入力します。
    *   本番環境: `http://localhost:3000`
    *   テスト環境: `http://localhost:3001`
3.  「Save」ボタンをクリックします。

### 2. APIサーバーの起動

meme-gtdのWeb UI/APIサーバーを起動しておいてください。

*   **本番環境**: `pnpm run server:start`
*   **テスト環境**: `pnpm server:dev`

### 3. 記事の保存

保存したいWebページを開き、ブラウザのツールバーにある「Meme GTD Saver」アイコンをクリックします。
ポップアップが表示され、「Save Article」ボタンをクリックすると、現在のページの内容（本文）が抽出され、Markdown形式でmeme-gtdに保存されます。

### 4. 保存結果の確認

Web UIの `http://localhost:3000/articles`（または設定したポート）にアクセスすると、保存された記事の一覧を確認できます。

## トラブルシューティング

- **"Error: Server error" と表示される場合**
  - APIサーバーが起動しているか確認してください。
  - 設定画面（オプション）で正しいAPI URLが設定されているか確認してください。
  - `http://localhost:3000/api/articles`（または設定したURL）にブラウザから直接アクセスできるか確認してください。

- **"Extraction failed" と表示される場合**
  - 特殊な構造のWebページなど、本文抽出に失敗する場合があります。コンソールログ（拡張機能のバックグラウンドページまたはコンテンツスクリプト）を確認してください。

- **拡張機能の更新**
  - コードを変更して再ビルドした後は、`chrome://extensions` ページで拡張機能の更新ボタン（回転矢印アイコン）をクリックしてリロードしてください。
